Brief Description of This Task

This is a new type of MBR Whitelisting that requires Technical Support to whitelist bills in order to allow them to export on the MBR.

Task Prerequisites

  • ✓ Access to GalantUT SQL Server
  • ✓ Read/Write Access to the ESIS DB on GalantUT
  • ✓ Store Procedure execution privileges on ESIS DB
  • ✓ DCNs requested for Rollback
  • ✓ Verification that all listed DCNs need to export with MSG TBR11

Keywords To Look For

  • MBR Validation
  • Whitelisting
  • Copmensable Body Part
  • TBR11

Identify The Ticket

Easy identification of these tickets can be made by either the title of the ticket and or the keywords that can be easily found in the ticket opening*.

Write Your SQL


                USE ESIS;

                DECLARE
                @UserName VARCHAR(20) = 'MBR-Rollback',
                @Reason VARCHAR(50) = 'Rollback Requested Ticket ',
                @Ticket VARCHAR(12) = --add the ticket number here,
                @DCNLIST DCNLIST;
                
                -- Populate @DCNLIST with the desired DCNs
                INSERT INTO @DCNLIST (DCN)
                SELECT documentcontrolnumber FROM bills
                WHERE documentcontrolnumber IN ( ); -- Add DCNs as comma separated list
                
                -- Insert DCNs from @DCNLIST into MBRSKipCompensableBodyPart
                INSERT INTO MBRSKipCompensableBodyPart (DCN)
                SELECT DCN FROM @DCNLIST;
                
                -- Log the rollback for each DCN
                INSERT INTO CEBRSLog (DCN, action, [UserName], DateTimeStamp)
                SELECT DCN, @Reason + @Ticket + ' Rolling Back from MBR tables', @UserName, GETDATE() FROM @DCNLIST;
                
                -- Execute the rollback stored procedure with @DCNLIST
                EXEC [MBRRollBackDCNs] @DCNLIST;

Pay special attention to the values that need to be updated when you run your sql. Both the Ticket variable (@Ticket) and the DCNs added to the DCN list need to be updated. Additionally, ensure that the DCNs are added as a comma separated list before attempting to run the script.
